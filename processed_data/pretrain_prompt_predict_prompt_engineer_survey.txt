ABSTRACT:
This paper surveys and organizes research works in a new paradigm in natural language processing called "prompt-based learning." Unlike traditional supervised learning, which trains a model P(y|x), prompt-based learning is based on language models that model the probability of text directly. The original input x is modified using a template into a textual string prompt x' that has unfilled slots, and the language model probabilistically fills them to obtain a final string x̂, from which the output y is derived.

Key advantages:
- Allows pre-training on massive amounts of raw text
- Enables few-shot or zero-shot learning by defining a new prompting function
- Adapts to new scenarios with few or no labeled data

The paper introduces basics of this paradigm, a unified mathematical notation, and organizes existing work along dimensions: choice of pre-trained models, prompts, and tuning strategies.

SECTION 1: TWO SEA CHANGES IN NLP

PARADIGM A — Fully Supervised Learning (Non-Neural Network):
- Task-specific model trained solely on input-output examples
- Relied heavily on feature engineering (domain knowledge to define/extract salient features)
- Examples: Lafferty et al. (2001), Guyon et al. (2002), Och et al. (2004), Zhang and Nivre (2011)

PARADIGM B — Fully Supervised Learning (Neural Network):
- Salient features learned jointly with model training
- Focus shifted to architecture engineering (designing suitable network architectures)
- Examples: Hochreiter & Schmidhuber (1997), Kalchbrenner et al. (2014), Chung et al. (2014), Kim (2014), Bahdanau et al. (2014), Vaswani et al. (2017)

FIRST SEA CHANGE (2017-2019) — Pre-train and Fine-tune Paradigm:
- Model with fixed architecture is pre-trained as a language model (LM)
- LM predicts probability of observed textual data
- Pre-trained LM adapted to downstream tasks by introducing additional parameters and fine-tuning
- Focus: objective engineering (designing training objectives for pre-training and fine-tuning)
- Examples: Radford & Narasimhan (2018), Peters et al. (2018), Dong et al. (2019), Yang et al. (2019), Lewis et al. (2020a)
- Example of objective engineering: Zhang et al. (2020a) — predicting salient sentences improves summarization

SECOND SEA CHANGE (as of 2021) — Pre-train, Prompt, and Predict:
- Downstream tasks reformulated to look like original LM training via textual prompts
- Instead of adapting LM to tasks (LM→Task), tasks are adapted to LMs (Task→LM)
- Example 1 (sentiment): "I missed the bus today. I felt so ___" — LM fills blank with emotion word
- Example 2 (translation): "English: I missed the bus today. French: ___" — LM fills blank with French
- Advantage: A single LM trained unsupervised can solve many tasks
- Catch: Requires prompt engineering to find best prompt

KEY CONCEPT — Prompt Engineering: Finding the most appropriate prompt to allow a LM to solve the task at hand.

SECTION 2.1: SUPERVISED LEARNING IN NLP

Traditional supervised learning: input x → model P(y|x; θ) → output y
- y can be a label, text, or other output
- Trained on datasets of input-output pairs

Example 1 — Text Classification:
- Sentiment analysis: x = "I love this movie." → y = ++ (from label set {++, +, ~, -, --})

Example 2 — Conditional Text Generation:
- Machine translation: x = "Hyvää huomenta." (Finnish) → y = "Good morning." (English)

SECTION 2.2: PROMPTING BASICS

Core problem with supervised learning: requires large supervised datasets for every task.

Prompt-based learning alternative:
- Learn a language model P(x; θ) that models probability of text x itself
- Use this to predict y, reducing need for large supervised datasets
- Basic prompting predicts highest-scoring ŷ in THREE STEPS: Prompt Addition → Answer Search → Answer Mapping

TERMINOLOGY (Table 2):
| Term | Notation | Example |
|------|----------|---------|
| Input | x | "I love this movie." |
| Output | y | ++ (very positive) |
| Prompting Function | f_prompt(x) | [X] Overall, it was a [Z] movie. |
| Prompt | x' | "I love this movie. Overall, it was a [Z] movie." |
| Filled Prompt | f_fill(x', z) | "I love this movie. Overall, it was a bad movie." |
| Answered Prompt | f_fill(x', z*) | "I love this movie. Overall, it was a good movie." |
| Answer | z | "good", "fantastic", "boring" |

Note: z* represents answers that correspond to true output y*

FOUR PARADIGMS IN NLP (Table 1):
| Paradigm | Engineering Type | Examples |
|----------|-----------------|---------|
| a. Fully Supervised (Non-Neural) | Features (word identity, POS, sentence length) | — |
| b. Fully Supervised (Neural) | Architecture (convolutional, recurrent, self-attentional) | — |
| c. Pre-train, Fine-tune | Objective (masked LM, next sentence prediction) | BERT, GPT |
| d. Pre-train, Prompt, Predict | Prompt (cloze, prefix) | GPT-3, etc. |

SECTION 2.2.1: PROMPT ADDITION

Process:
1. Apply a template — textual string with two slots: [X] (input slot) and [Z] (answer slot)
2. Fill slot [X] with input text x

Sentiment example:
- Input x = "I love this movie."
- Template = "[X] Overall, it was a [Z] movie."
- Prompt x' = "I love this movie. Overall, it was a [Z] movie."

Translation example:
- Template = "Finnish: [X] English: [Z]"

TWO PROMPT VARIETIES:
1. Cloze prompt: slot [Z] appears in the MIDDLE of text
2. Prefix prompt: input text comes entirely BEFORE [Z] (slot at end)

IMPORTANT NOTES:
(1) Templates can have empty slots in middle (cloze) or at end (prefix)
(2) Template words are not necessarily natural language — can be virtual words (numeric IDs) embedded in continuous space
(3) Number of [X] and [Z] slots can be flexibly changed per task

EXAMPLES (Table 3 — Inputs, Templates, Answers for different tasks):

Text Classification — Sentiment:
  Input: "I love this movie."
  Template: "[X] The movie is [Z]."
  Answer: great, fantastic, ...

Text Classification — Topics:
  Input: "He prompted the LM."
  Template: "[X] The text is about [Z]."
  Answer: sports, science, ...

Text Classification — Intention:
  Input: "What is taxi fare to Denver?"
  Template: "[X] The question is about [Z]."
  Answer: quantity, city, ...

Text-span Classification — Aspect Sentiment:
  Input: "Poor service but good food."
  Template: "Bad [X] What about service? [Z]."
  Answer: Terrible, ...

Text-pair Classification — NLI:
  Input: [X1] "An old man with ...", [X2] "A man walks ..."
  Template: "[X1]? [Z], [X2]"
  Answer: Yes, No, ...

Tagging — NER:
  Input: [X1] "Mike went to Paris.", [X2] "Paris"
  Template: "[X1][X2] is a [Z] entity."
  Answer: organization, location, ...

Text Generation — Summarization:
  Input: "Las Vegas police ..."
  Template: "[X] TL;DR: [Z]"
  Answer: "The victim ...", "A woman ...", ...

Text Generation — Translation:
  Input: "Je vous aime."
  Template: "French: [X] English: [Z]"
  Answer: "I love you.", "I fancy you.", ...

SECTION 2.2.2: ANSWER SEARCH

- Define Z = set of permissible values for z
- Z can range from entire language (generative tasks) to small subset (classification)
- Example classification: Z = {"excellent", "good", "OK", "bad", "horrible"} for Y = {++, +, ~, -, --}
- f_fill(x', z) = fills [Z] position in prompt x' with potential answer z
- Search function: ẑ = search_{z∈Z} P(f_fill(x', z); θ)
- Search can be argmax or sampling

SECTION 2.2.3: ANSWER MAPPING

- Map highest-scoring answer ẑ to highest-scoring output ŷ
- Trivial in generation tasks (answer = output)
- Non-trivial when multiple answers map to same output (e.g., "excellent", "fabulous", "wonderful" all map to "++")

SECTION 3: PRE-TRAINED LANGUAGE MODELS

Overview dimensions: main training objective, type of text noising, auxiliary training objective, attention mask, typical architecture, preferred application scenarios.

SECTION 3.1: TRAINING OBJECTIVES

Standard Language Model (SLM):
- Trains model to optimize probability P(x) of text from training corpus (Radford et al., 2019)
- Text predicted in autoregressive fashion, one token at a time (usually left to right)

Denoising Objectives:
- Apply noising function x̃ = f_noise(x) to input
- Try to predict original input P(x|x̃)
- Two flavors:
  (a) Corrupted Text Reconstruction (CTR): Loss calculated only over noised parts
  (b) Full Text Reconstruction (FTR): Loss calculated over entirety of input (Lewis et al., 2020a)

MATCHING OBJECTIVES TO TASKS:
- Left-to-right autoregressive LMs → particularly suitable for prefix prompts
- Reconstruction objectives → more suitable for cloze prompts
- Standard LM and FTR objectives → more suitable for text generation tasks
- Any of the objectives → can be used for classification

SECTION 3.2: NOISING FUNCTIONS

Types of noise (Table 4 — Detailed examples):

Operation | Element | Original Text | Corrupted Text
---------|---------|---------------|---------------
Mask | one token | Jane will move to New York . | Jane will [Z] to New York .
Mask | two tokens | Jane will move to New York . | Jane will [Z] [Z] New York .
Mask | one entity | Jane will move to New York . | Jane will move to [Z] .
Replace | one token | Jane will move to New York . | Jane will move [X] New York .
Replace | two tokens | Jane will move to New York . | Jane will move [X] [Y] York .
Replace | one entity | Jane will move to New York . | Jane will move to [X] .
Delete | one token | Jane will move to New York . | Jane move to New York .
Delete | two tokens | Jane will move to New York . | Jane to New York .
Permute | token | Jane will move to New York . | New York . Jane will move to
Rotate | none | Jane will move to New York . | to New York . Jane will move
Concatenation | two languages | Jane will move to New York . | Jane will move to New York . [/s] 简将搬到纽约。

NOISING TYPES:
- Masking (Devlin et al., 2019): Replace token/span with [MASK]; can be random or knowledge-targeted (e.g., masking entities)
- Replacement (Raffel et al., 2020): Replace token/span with another token or piece of information (e.g., image region)
- Deletion (Lewis et al., 2020a): Delete tokens/spans without adding [MASK]; used with FTR loss
- Permutation (Liu et al., 2020a): Divide text into spans, permute into new text

SECTION 3.3: DIRECTIONALITY OF REPRESENTATIONS

Two main ways to calculate representations:

1. Left-to-Right:
   - Representation of each word based on itself and all PREVIOUS words
   - Example: "This is a good movie" — "good" based on preceding words only
   - Used for standard LM objectives and output side of FTR objectives

2. Bidirectional:
   - Representation of each word based on ALL words in sentence (both left and right)
   - Example: "good" influenced by all words including following "movie"

ADDITIONAL:
- Models can mix both strategies (Dong et al., 2019; Bao et al., 2020)
- Representations can be calculated in randomly permuted order (Yang et al., 2019)
- Implemented via attention masking in Transformer architecture (Vaswani et al., 2017)

ATTENTION MASK PATTERNS (Figure 2):
(a) Full — all positions can attend to all others (bidirectional)
(b) Diagonal — each position only attends to itself and previous (left-to-right)
(c) Mixture — combination of full and diagonal

SECTION 3.4: TYPICAL PRE-TRAINING METHODS

Pre-training architectures summarized (Table 5):

--------|----------|----------|------------|----------|----------|------------|------------

SECTION 3.4.1: LEFT-TO-RIGHT LANGUAGE MODEL

- Variety of auto-regressive LM
- Predicts upcoming words or assigns probability P(x) to sequence x = x1,...,xn
- Chain rule (left-to-right): P(x) = P(x1) × ... × P(xn|x1...xn-1)
- Standard since Markov (1913); used in count-based and neural forms
- Representative modern models: GPT-3 (Brown et al., 2020), GPT-Neo (Black et al., 2021)
- Note: right-to-left variant: P(xi|xi+1,...,xn)
- L2R LMs widely used as backbone for prompting (Radford et al., 2019; Brown et al., 2020)
- Large models (PanGu-α, Ernie-3) often not available publicly, making fine-tuning infeasible

SECTION 3.4.2: MASKED LANGUAGE MODELS (MLM)

- Bidirectional objective function for representation learning
- Predicts masked text pieces based on surrounding context
- Formula: P(xi|x1,...,xi-1, xi+1,...,xn) — probability of word xi given surrounding context
- Devlin et al. (2019)

Representative models: BERT (Devlin et al., 2019), ERNIE (Zhang et al., 2019; Sun et al., 2019b)

Best suited for:
- Natural language understanding tasks (text classification, NLI, extractive QA)
- Tasks easily reformulated as cloze problems (consistent with MLM training)
- Methods combining prompting with fine-tuning

SECTION 3.4.3: PREFIX AND ENCODER-DECODER

Both architectures:
1. Encode source x using encoder with fully-connected mask
2. Decode target y auto-regressively (left to right)

PREFIX LANGUAGE MODEL:
- Left-to-right LM that decodes y conditioned on prefixed sequence x
- x encoded by same model parameters with fully-connected mask
- Usually uses CTR objective over x, plus standard conditional LM objective over y
- Examples: UniLM 1-2 (Dong et al., 2019; Bao et al., 2020), ERNIE-M (Ouyang et al., 2020)

ENCODER-DECODER MODEL:
- Uses left-to-right LM to decode y conditioned on separate encoder for x
- Encoder and decoder parameters NOT shared
- Diverse types of noising can be applied to x
- Examples: T5 (Raffel et al., 2020), BART (Lewis et al., 2020a), MASS (Song et al., 2019)

APPLICABILITY:
- Naturally suited for text generation (translation, summarization) with or without prompting
- Prompting methods broaden applicability to non-generation tasks:
  - Information extraction (Cui et al., 2021)
  - Question answering (Khashabi et al., 2020)
  - Text generation evaluation (Yuan et al., 2021b)
- Prompting makes models like BART applicable to NER (previously less used)
- Prompting breaks difficulty of unified modeling across different tasks (Khashabi et al., 2020)

SECTION 4: PROMPT ENGINEERING

Definition: Process of creating a prompting function f_prompt(x) that results in most effective downstream performance.
Focus: Prompt template engineering — human or algorithm searches for best template per task.

SECTION 4.1: PROMPT SHAPE

Two main varieties:
1. Cloze prompts (Petroni et al., 2019; Cui et al., 2021): Fill in blanks of a textual string
2. Prefix prompts (Li & Liang, 2021; Lester et al., 2021): Continue a string prefix

Choosing between them:
- Generation tasks / autoregressive LMs → prefix prompts (match left-to-right nature)
- Masked LMs → cloze prompts (closely match pre-training task)
- Full text reconstruction models → versatile, can use either
- Tasks with multiple inputs (e.g., text pair classification) → templates with [X1], [X2], etc.

SECTION 4.2: MANUAL TEMPLATE ENGINEERING

- Most natural approach: manually create intuitive templates via human introspection
- Seminal example: LAMA dataset (Petroni et al., 2019) — manually created cloze templates to probe knowledge
- Brown et al. (2020) — manually crafted prefix prompts for QA, translation, commonsense reasoning
- Schick & Schütze (2020, 2021a,b) — pre-defined templates for few-shot text classification and generation

SECTION 4.3: AUTOMATED TEMPLATE LEARNING

Issues with manual templates:
1. Time-consuming and requires expertise (especially for complex tasks like semantic parsing)
2. Even experienced designers may fail to find optimal prompts (Jiang et al., 2020c)

TWO TYPES:
1. Discrete prompts (hard prompts): actual text strings
2. Continuous prompts (soft prompts): described directly in embedding space

Also: static vs. dynamic prompting functions (same template per input vs. custom per input)

SECTION 4.3.1: DISCRETE PROMPTS

D1 — Prompt Mining (Jiang et al., 2020c — MINE approach):
- Mining-based method using training inputs x and outputs y
- Scrapes large text corpus (e.g., Wikipedia) for strings containing x and y
- Finds middle words or dependency paths between inputs and outputs
- Frequent middle words/paths serve as templates: "[X] middle words [Z]"

D2 — Prompt Paraphrasing:
- Takes existing seed prompt, paraphrases to create candidate prompts
- Selects prompt with highest training accuracy
- Methods: round-trip translation (Jiang et al., 2020c), thesaurus replacement (Yuan et al., 2021b), neural prompt rewriter (Haviv et al., 2021)
- Haviv et al. (2021): paraphrasing done after input x is inserted, different paraphrase per individual input

D3 — Gradient-based Search:
- Wallace et al. (2019a): gradient-based search over actual tokens to find sequences triggering desired predictions
- Iterative, stepping through tokens in the prompt
- Shin et al. (2020): AutoPrompt — automatically searches for template tokens using downstream training samples; strong performance

D4 — Prompt Generation:
- Treat prompt generation as text generation task
- Gao et al. (2021): Use seq2seq T5 to generate template tokens
  - T5 pre-trained on filling missing spans
  - Specify positions to insert template tokens
  - Provide training samples for T5 to decode template tokens
- Ben-David et al. (2021): Train T5 to generate domain-relevant features (DRFs) — keywords characterizing domain — then concatenate with input to form template

D5 — Prompt Scoring:
- Davison et al. (2019): knowledge base completion task
- Hand-craft set of template candidates
- Fill input and answer slots to form filled prompts
- Use unidirectional LM to score filled prompts
- Select template with highest LM probability (custom per input)

SECTION 4.3.2: CONTINUOUS PROMPTS (SOFT PROMPTS)

Key insight: Prompts don't need to be human-interpretable. They can exist in embedding space.

Two constraints removed:
1. Embeddings of template words need NOT be embeddings of natural language words
2. Template need NOT be parameterized by the pre-trained LM's parameters; templates have their OWN parameters tuned on downstream training data

C1 — Prefix Tuning (Li & Liang, 2021):
- Prepends sequence of continuous task-specific vectors to input
- LM parameters kept FROZEN
- Optimizes trainable prefix matrix M_φ:
  max_φ log P(y|x; θ; φ) = max_φ Σ_yi log P(yi|h<i; θ; φ)
- h<i = concatenation of all neural network layers at time step i
- If within prefix: h_i = M_φ[i]; otherwise computed by frozen LM
- Observation: Continuous prefix learning more sensitive to initialization in low-data settings than discrete prompts
- Lester et al. (2021) — similar but adds fewer parameters (no additional tunable params within each network layer)
- Tsimpoukelli et al. (2021) — trains vision encoder to encode image into embeddings that prompt frozen LM; enables few-shot learning for vision-language tasks; prefix is sample-dependent (image representation)

C2 — Tuning Initialized with Discrete Prompts:
- Initialize continuous prompt search using discrete prompt search results
- Zhong et al. (2021b): Define template via discrete search (AUTOPROMPT), initialize virtual tokens from discovered prompt, then fine-tune embeddings for task accuracy
- Initializing with manual templates provides better starting point
- Qin & Eisner (2021): Learn mixture of soft templates per input; weights and parameters jointly learned from training; initial templates from manual or prompt mining
- Hambardzumyan et al. (2021): Use continuous template whose shape follows a manual template

C3 — Hard-Soft Prompt Hybrid Tuning:
- Insert tunable embeddings into hard prompt template
- Liu et al. (2021b) — P-tuning:
  - Continuous prompts learned by inserting trainable variables into embedded input
  - Prompt embeddings represented as output of BiLSTM (Graves et al., 2013) to account for interaction between prompt tokens
  - Introduces task-related anchor tokens (e.g., "capital" in relation extraction) — NOT tuned during training
- Han et al. (2021) — PTR (Prompt Tuning with Rules):
  - Manually crafted sub-templates composed via logic rules
  - Inserts virtual tokens whose embeddings can be tuned together with LM parameters
  - Template tokens contain both actual and virtual tokens
  - Effective for relation classification

SECTION 5: ANSWER ENGINEERING

Definition: Searches for answer space Z and map to original output Y that results in effective predictive model.
Two dimensions: Answer shape and Answer design method.

SECTION 5.1: ANSWER SHAPE

Answer shape granularity options:

1. Tokens: One token from pre-trained LM's vocabulary (or subset)
2. Span: Short multi-token span — usually used with cloze prompts
3. Sentence: A sentence or document — commonly used with prefix prompts

Choosing answer shape:
- Token/text-span: Widely used in classification (sentiment classification — Yin et al., 2019), relation extraction (Petroni et al., 2019), NER (Cui et al., 2021)
- Longer phrasal/sentential: Language generation tasks (Radford et al., 2019), multiple-choice QA (Khashabi et al., 2020)

SECTION 5.2: ANSWER SPACE DESIGN METHODS

SECTION 5.2.1: MANUAL DESIGN

Unconstrained Spaces:
- Answer space Z = all tokens (Petroni et al., 2019), fixed-length spans (Jiang et al., 2020a), or token sequences (Radford et al., 2019)
- Usually map z to y via identity mapping

Constrained Spaces:
- For limited label spaces (text classification, entity recognition, multiple-choice QA)
- Yin et al. (2019): Manually design word lists per topic ("health", "finance", "politics", "sports"), emotions ("anger", "joy", "sadness", "fear")
- Cui et al. (2021): Manually design lists like "person", "location" for NER
- Zweig et al. (2012): Use LM to calculate probability among multiple choices for multiple-choice QA

SECTION 5.2.2: DISCRETE ANSWER SEARCH

Answer Paraphrasing:
- Start with initial answer space Z0, expand via paraphrasing
- Jiang et al. (2020b): P(y|x) = Σ_{z∈para(z0)} P(z|x)
- Uses back-translation for paraphrasing (translate to another language then back)

Prune-then-Search:
- Generate initial pruned answer space of plausible answers Z0
- Algorithm further searches over Z0 for final answers
- "Verbalizer": function from label y to single answer token z (Schick & Schütze, 2021a)
- Schick & Schütze (2021a); Schick et al. (2020): Find tokens with ≥2 alphabetic characters, frequent in large unlabeled dataset; iteratively compute word's suitability as representative answer
- Shin et al. (2020): Learn logistic classifier using contextualized representation of [Z] token; select top-k tokens by probability score
- Gao et al. (2021): 
  1. Prune: Select top-k vocabulary words by generation probability at [Z] position
  2. Further prune by zero-shot accuracy on training samples
  3. Search: Fine-tune LM with fixed templates + every answer mapping; select best by development set accuracy

Label Decomposition:
- Chen et al. (2021b): For relation extraction, automatically decompose each relation label into constituent words
- Example: per:city_of_death → {person, city, death}
- Probability = sum of each token's probability

SECTION 5.2.3: CONTINUOUS ANSWER SEARCH

- Very few works explore soft answer tokens optimizable via gradient descent
- Hambardzumyan et al. (2021): Assign virtual token per class label; optimize token embedding jointly with prompt token embeddings
- Since tokens optimized directly in embedding space, they don't use LM's learned embeddings — learn from scratch per label

SECTION 6: MULTI-PROMPT LEARNING

Motivation: Using multiple prompts can further improve efficacy of prompting methods.
(Single prompt engineering covered in §4.)

FOUR MULTI-PROMPT METHODS:
1. Prompt Ensembling
2. Prompt Augmentation
3. Prompt Composition
4. Prompt Decomposition

SECTION 6.1: PROMPT ENSEMBLING

Definition: Using multiple unanswered prompts for an input at inference time to make predictions.
Prompts can be discrete or continuous.

Benefits:
1. Leverage complementary advantages of different prompts
2. Alleviate cost of prompt engineering (choosing single best prompt is challenging)
3. Stabilize performance on downstream tasks

Combination methods:

Uniform Averaging:
- P(z|x) := (1/K) Σ_i P(z|f_prompt,i(x))
- Jiang et al. (2020c): Filter by top-K accuracy, use average log probabilities
- Schick & Schütze (2021a): Simple average for ensemble annotation of unlabeled data
- Yuan et al. (2021b): Average of generation scores across prompts

Weighted Averaging:
- Each prompt associated with a weight (pre-specified or optimized)
- Jiang et al. (2020c): Learn weight per prompt by maximizing P(target) over training data
- Qin & Eisner (2021): Same approach + data-dependent weighting based on input probability in each prompt
- Schick & Schütze (2021a,b): Weight proportional to training set accuracy before training

Majority Voting:
- For classification tasks, combine results from different prompts via majority vote
- Lester et al. (2021); Hambardzumyan et al. (2021)

Knowledge Distillation:
- Schick & Schütze (2021a,b, 2020): Train separate model per template-answer pair; ensemble annotates unlabeled dataset; final model distills knowledge from annotated data
- Gao et al. (2021): Similar ensemble on auto-generated templates

Prompt Ensembling for Text Generation:
- Simple: P(zt|x, z<t) := (1/K) Σ_i P(zt|f_prompt,i(x), z<t)
- Schick & Schütze (2020): Decode with each model separately; score each generation by averaging probability across all models

SECTION 6.2: PROMPT AUGMENTATION

Also called: Demonstration learning (Gao et al., 2021)

Definition: Provide a few additional answered prompts to demonstrate how LM should answer the actual prompt.

Example:
- Basic: "China's capital is [Z]."
- Augmented: "Great Britain's capital is London. Japan's capital is Tokyo. China's capital is [Z]."

Leverages ability of strong LMs to learn repetitive patterns (Brown et al., 2020).

Two key challenges:

1. Sample Selection:
   - Choice of examples can result in very different performance (near SOTA to near random — Lu et al., 2021)
   - Gao et al. (2021); Liu et al. (2021a): Use sentence embeddings to sample examples close to input
   - Mishra et al. (2021): Provide positive and negative samples (highlighting things to avoid)

2. Sample Ordering:
   - Lu et al. (2021): Order of answered prompts plays important role in performance; propose entropy-based methods to score different candidate permutations
   - Kumar & Talukdar (2021): Search for good permutation of training examples; learn separator token between prompts

Relation to retrieval-based methods:
- Retrieval provides more textual context to improve performance (Guu et al., 2018)
- Prompt augmentation also leverages template and answer (not just larger context)

SECTION 6.3: PROMPT COMPOSITION

Definition: For composable tasks, use multiple sub-prompts (one per subtask), then define a composite prompt.

Illustration (Figure 4c):
- Relation extraction = entity recognition + relation classification
- Create sub-prompts for entity types, then compose via logic rules
- Han et al. (2021) — PTR: multiple manually created sub-prompts for entity recognition and relation classification; composed via logic rules for relation extraction; also incorporates entity type information

SECTION 6.4: PROMPT DECOMPOSITION

Definition: For tasks requiring multiple predictions per sample (e.g., sequence labeling), break holistic prompt into sub-prompts and answer each separately.

Example — NER (Cui et al., 2021):
- Input converted into set of text spans
- Model prompted to predict entity type per span (including "Not an Entity")
- Different prompt per span; each predicted separately
- Difficult to predict all span types simultaneously due to large number of spans

FIGURE 4 SUMMARY — Four multi-prompt strategies:
(a) Prompt Ensembling: multiple prompts → aggregate predictions
(b) Prompt Augmentation: add answered prompts as demonstrations before actual prompt
(c) Prompt Composition: combine sub-prompts for sub-tasks into composite prompt
(d) Prompt Decomposition: break single complex prompt into independent sub-prompts

SECTION 7: TRAINING STRATEGIES FOR PROMPTING METHODS

SECTION 7.1: TRAINING SETTINGS

Zero-shot: No training data for the task; LM applied as-is using prompt.
Few-shot: Small number of training examples.
Full-data: Large number of training examples.

NOTE: Many prompt engineering methods (§4) use annotated training samples for constructing/validating prompts even in "zero-shot" settings — Perez et al. (2021) argue this is not true zero-shot.

SECTION 7.2: PARAMETER UPDATE METHODS

Two types of parameters:
1. Pre-trained model parameters (LM)
2. Prompt-related parameters

TABLE 6 — Five Tuning Strategies:

Strategy | LM Params | Additional Prompt Params? | Prompt Params Tuned? | Example
---------|-----------|--------------------------|---------------------|--------
Promptless Fine-tuning | Tuned | No | — | ELMo, BERT, BART
Tuning-free Prompting | Frozen | No | — | GPT-3, AutoPrompt, LAMA
Fixed-LM Prompt Tuning | Frozen | Yes | Tuned | Prefix-Tuning, Prompt-Tuning
Fixed-prompt LM Tuning | Tuned | No | — | PET-TC, PET-Gen, LM-BFF
Prompt+LM Fine-tuning | Tuned | Yes | Tuned | PADA, P-Tuning, PTR

SECTION 7.2.1: PROMPTLESS FINE-TUNING

- Pre-train and fine-tune WITHOUT prompts
- All (or some) LM parameters updated via gradients from downstream samples
- Examples: BERT [32], RoBERTa [105]
- Advantages: Simplicity, no prompt design needed; can fit larger training datasets
- Disadvantages: May overfit or learn unstably on small datasets; prone to catastrophic forgetting (McCloskey & Cohen, 1989)

SECTION 7.2.2: TUNING-FREE PROMPTING

- Directly generates answers WITHOUT changing LM parameters; based only on prompt
- Can combine with prompt augmentation (in-context learning; Brown et al., 2020)
- Examples: LAMA [133], GPT-3 [16]
- Advantages: Efficient (no parameter update); no catastrophic forgetting; applicable in zero-shot settings
- Disadvantages: Heavy prompt engineering needed for high accuracy; in-context learning is slow at test time with many demonstrations; can't use large training datasets easily

SECTION 7.2.3: FIXED-LM PROMPT TUNING

- Introduces additional prompt-relevant parameters
- Updates ONLY prompt parameters using downstream supervision
- Entire pre-trained LM UNCHANGED
- Examples: Prefix-Tuning [96], WARP [55]
- Advantages: Retains LM knowledge; suitable for few-shot scenarios; often superior to tuning-free
- Disadvantages: Not applicable in zero-shot; limited representation power in large-data settings; requires prompt engineering via hyperparameter/seed prompt choice; prompts usually not human-interpretable

SECTION 7.2.4: FIXED-PROMPT LM TUNING

- Tunes LM parameters (standard pre-train and fine-tune style) but uses FIXED prompt parameters
- Most natural: discrete textual template applied to every training/test example
- Examples: PET-TC [153], PET-Gen [152], LM-BFF [46]
- Logan IV et al. (2021): Null prompt — directly concatenate input and mask "[X][Z]" without template words; competitive accuracy
- Advantages: Prompt/answer engineering specifies task more completely; efficient learning, especially few-shot
- Disadvantages: Prompt/answer engineering still required; LM fine-tuned on one task may not generalize to another

SECTION 7.2.5: PROMPT+LM TUNING

- Prompt-relevant parameters fine-tuned TOGETHER with all or some LM parameters
- Examples: PADA [8], P-Tuning [103]
- Similar to standard pre-train/fine-tune, but prompt provides additional bootstrapping
- Advantages: Most expressive; likely suitable for high-data settings
- Disadvantages: Requires training and storing all parameters; may overfit to small datasets

SECTION 8: APPLICATIONS

SECTION 8.1: KNOWLEDGE PROBING

Factual Probing (a.k.a. fact retrieval):
- One of earliest prompt applications
- Goal: Quantify how much factual knowledge LM's internal representations contain
- LM parameters usually fixed; knowledge retrieved by transforming input to cloze prompt
- Datasets: LAMA (Petroni et al., 2019), X-FACTR (Jiang et al., 2020a)
- Both discrete (Petroni et al., 2019, 2020; Jiang et al., 2020c, a; Haviv et al., 2021; Shin et al., 2020; Perez et al., 2021) and continuous (Qin & Eisner, 2021; Liu et al., 2021b; Zhong et al., 2021b) template methods explored
- Also prompt ensemble learning (Jiang et al., 2020c; Qin & Eisner, 2021)

Linguistic Probing:
- Probe capabilities: analogies, negations, semantic role sensitivity, semantic similarity, rare word understanding
- Elicited by presenting tasks as natural language sentences for LM completion
- Sources: Brown et al. (2020), Ettinger (2020), Sun et al. (2021), Schick & Schütze (2020)

SECTION 8.2: CLASSIFICATION-BASED TASKS

Overview:
- Widely explored; prompt templates relatively easy to construct
- Key: reformulating task as appropriate prompt
- Example: Yin et al. (2019) — "the topic of this document is [Z]." fed into masked LM

Text Classification:
- Most work uses cloze prompts
- Both prompt engineering and answer engineering extensively explored
- Most explore few-shot setting with "fixed-prompt LM Tuning" strategy (§7.2.4)

Natural Language Inference (NLI):
- Goal: Predict relationship (e.g., entailment) between two sentences
- Cloze prompts commonly used (Schick & Schütze, 2021a)
- Template search in few-shot setting; answer space Z manually pre-selected from vocabulary

SECTION 8.3: INFORMATION EXTRACTION

Relation Extraction:
- Goal: Predict relation between two entities in a sentence
- Chen et al. (2021b): First explored fixed-prompt LM Tuning for relation extraction
- Key challenges vs. classification:
  1. Larger label space (e.g., 80 relations vs. 2 for binary sentiment) → harder answer engineering
  2. Different input tokens have different importance (entity mentions more salient) → not reflected in standard templates
- Solutions:
  - Adaptive answer selection for label space challenge
  - Task-oriented prompt template with special markers [E] to highlight entity mentions
- Han et al. (2021): Incorporate entity type information via multiple prompt composition techniques (PTR)

Semantic Parsing:
- Goal: Generate structured meaning representation from natural language
- Shin et al. (2021): Few-shot semantic parsing via:
  1. Framing as paraphrasing task (Berant & Liang, 2014)
  2. Constraining decoding to only allow valid output according to grammar
- In-context learning setting; choose answered prompts semantically close to test example
- Effective for semantic parsing using pre-trained LMs

Named Entity Recognition (NER):
- Goal: Identify named entities (person, location, etc.) in a sentence
- Challenges for prompt-based learning:
  1. Each prediction unit is a token/span (not entire input like classification)
  2. Latent relationships between token labels in context
- Cui et al. (2021): Template-based NER using BART
  - Enumerate text spans
  - Consider generation probability of each type within manually crafted templates
  - Example: "Mike went to New York yesterday" → to determine "Mike" type: "Mike is a [Z] entity"
  - Answer space: "person", "organization", etc.

SECTION 8.4: "REASONING" IN NLP

Debate: Whether neural networks truly reason or only memorize patterns (Arpit et al., 2017; Niven & Kao, 2019).

Commonsense Reasoning:
- Datasets: Winograd Schemas (Levesque et al., 2012), etc.
- Winograd Schema example: "The trophy doesn't fit into the brown suitcase because it is too large." — identify antecedent of "it"
- Approach: Replace "it" with candidates in sentence; calculate probability; choose highest (Trinh & Le, 2018)
- Sentence completion example: "Eleanor offered coffee. She realized she didn't have a clean [Z]." Choices: "cup", "bowl", "spoon"
- Approach: Score generation probability of each candidate (Ettinger, 2020)

Mathematical Reasoning:
- Pre-trained embeddings can perform simple operations (addition, subtraction for small digits)
- Fail with larger numbers (Naik et al., 2019; Wallace et al., 2019b; Brown et al., 2020)
- Reynolds & McDonell (2021): Improve LM performance on complex math via serializing reasoning

SECTION 8.5: QUESTION ANSWERING

Formats:
- Extractive QA: Identify content from context (e.g., SQuAD — Rajpurkar et al., 2016)
- Multiple-choice QA: Pick from several choices (e.g., RACE — Lai et al., 2017)
- Free-form QA: Arbitrary text response (e.g., NarrativeQA — Kočiský et al., 2018)

Benefit of LM+prompting: Different QA formats solved within same framework.

Key work:
- Khashabi et al. (2020) — UnifiedQA: Reformulate many QA tasks as text generation, fine-tuning seq2seq models (T5) with appropriate prompts
- Jiang et al. (2020b): Observe that probabilities from pre-trained models on QA tasks are not very predictive of correctness

SECTION 8.6: TEXT GENERATION

- Prefix prompts + autoregressive LMs
- Radford et al. (2019): Impressive generation performance (summarization, translation) using prompts like "translate to french, [X], [Z]"
- Brown et al. (2020): In-context learning (§7.2.2) for text generation — manual templates + multiple answered prompts
- Schick & Schütze (2020): Fixed-prompt LM tuning (§7.2.4) for few-shot text summarization with manual templates
- Li & Liang (2021): Fixed-LM prompt tuning (§7.2.3) for summarization and data-to-text — learnable prefix tokens, frozen LM
- Dou et al. (2021): Prompt+LM tuning (§7.2.5) for summarization — learnable prefix prompts initialized by guidance signals; both prompt and LM parameters updated

SECTION 8.7: AUTOMATIC EVALUATION OF TEXT GENERATION

- Yuan et al. (2021b) — BARTScore: Prompt learning for automated evaluation of generated texts
- Conceptualize evaluation as text generation problem
- Use pre-trained seq2seq model with prefix prompts
- Adding "such as" to translated text improves correlation significantly on German-English MT evaluation

SECTION 8.8: MULTI-MODAL LEARNING

- Tsimpoukelli et al. (2021) — Frozen: Extends prompt learning to vision-and-language
- Strategy: Fixed-LM prompt tuning + prompt augmentation
- Represent each image as sequence of continuous embeddings
- Frozen pre-trained LM prompted with image prefix to generate captions
- Demonstrates few-shot learning ability: learn words for new objects and novel visual categories with few demonstrations

SECTION 8.9: META-APPLICATIONS

Domain Adaptation:
- Ben-David et al. (2021) — PADA: Self-generated domain-related features (DRFs) concatenated with input
- Train T5 to generate unique domain-relevant keywords per input
- Perform sequence tagging as seq2seq problem

Debiasing:
- Schick et al. (2021): LMs perform self-diagnosis and self-debiasing via instructions
- Self-diagnosis: "The following text contains violence. [X][Z]" → fill [X] with text → check P("Yes") vs P("No")
- Self-debiasing: Combine original P(xt|x<t; θ) with diagnosis-conditioned P(xt|[x<t; x_diagnosis]; θ) to suppress undesired attribute

Dataset Construction:
- Schick & Schütze (2021): Pre-trained LMs generate datasets from instructions
- Example: "Write two sentences that mean the same thing. [X][Z]" → generate semantically similar sentence pairs

DETAILED TOPIC CONNECTIONS:

Ensemble Learning:
- Prompt ensembling doesn't require retraining the model multiple times
- Discrete prompts can be changed during inference stage (Jiang et al., 2020c)

Few-shot Learning:
- Methods: model-agnostic meta-learning (Finn et al., 2017b), embedding learning (Bertinetto et al., 2016), memory-based learning (Kaiser et al., 2017)
- Prompt augmentation = priming-based few-shot learning (Kumar & Talukdar, 2021)
- Advantage: directly prepends labeled samples to current sample; elicits knowledge from pre-trained LMs even without parameter tuning

Larger-context Learning:
- Cao et al. (2018): Augment input with retrieved training set context
- Guu et al. (2020): External data sources
- Prompt augmentation adds relevant labeled samples; larger-context learning does not necessarily use labeled data

Query Reformulation:
- Commonalities: Both aim to better use existing knowledge bases; knowledge base is often black-box
- Differences: Traditional knowledge base = search engine or QA system; prompt-based = language model
- Prompt learning changes form of task (classification → cloze problem), adding complexity
- Methodologies borrowable: decomposing query into sub-queries (Nogueira et al., 2019) ≈ prompt decomposition

QA-based Task Formulation:
- Kumar et al. (2016), McCann et al. (2018): Earlier works unifying NLP tasks into QA framework
- Explored in IE (Li et al., 2020; Wu et al., 2020) and text classification (Chai et al., 2020)
- Similar to prompting but key difference: prompting focuses on how to better use knowledge in pre-trained LMs

Controlled Generation (continued):
- Guidance signals: style tokens (Sennrich et al., 2016b; Fan et al., 2018), length specifications (Kikuchi et al., 2016), domain tags (Chu et al., 2017), keywords (Saito et al., 2020), relation triples (Zhu et al., 2020), highlighted phrases/sentences (Grangier & Auli, 2018; Liu et al., 2021c)
- Commonalities with prompting:
  (1) Both add extra information to input; additional signals are often learnable parameters
  (2) If controlled generation uses seq2seq LMs (e.g., BART) → can be seen as prompt learning with input-dependent prompts + prompt+LM fine-tuning strategy (e.g., GSum — Dou et al., 2021)
- Differences:
  (1) Controlled generation controls style/content while task stays same; prompting specifies the task itself
  (2) Most prompt learning uses dataset/task-level prompts; controlled generation commonly uses input-dependent ones → opportunity for cross-pollination

Supervised Attention:
- Liu et al. (2017b): Provides explicit supervision over attention to avoid overfitting to artifacts
- Both prompt learning and supervised attention: extract salient information with clues provided separately
- Supervised attention uses additional loss functions to predict gold attention on manually labeled corpus
- Prompt learning could borrow ideas from this literature

Data Augmentation:
- Scao & Rush (2021): Adding prompts ≈ adding 100s of data points on average across classification tasks
- Suggests prompt usage for downstream task ≈ implicit data augmentation

SECTION 10: CHALLENGES

SECTION 10.1: PROMPT DESIGN

Tasks beyond Classification and Generation:
- Most existing work revolves around text classification or generation
- Information extraction and text analysis tasks: prompt design is less straightforward
- Future approach: Either reformulate as classification/generation, or design effective answer engineering for structured outputs in textual format

Prompting with Structured Information:
- Inputs with tree, graph, table, or relational structure — how to express in prompt/answer engineering is a major challenge
- Chen et al. (2021b): Prompts with additional marks to encode lexical information (e.g., entity markings)
- Aghajanyan et al. (2021) — HTLM: Structured prompts based on HTML for fine-grained web text generation
- Moving beyond this to more complicated structure: largely unexplored

Entanglement of Template and Answer:
- Performance depends on BOTH template and answer
- How to simultaneously search/learn best combination remains challenging
- Current works: typically select answers BEFORE selecting template (Gao et al., 2021; Shin et al., 2020)
- Hambardzumyan et al. (2021): Initial potential of simultaneously learning both

SECTION 10.2: ANSWER ENGINEERING

Many-class and Long-answer Classification:
(a) Many classes → selecting appropriate answer space = difficult combinatorial optimization problem
(b) Multi-token answers → how to best decode using LMs remains unknown (some methods: Jiang et al., 2020a)

Multiple Answers for Generation Tasks:
- Qualified answers can be semantically equivalent but syntactically diverse
- Almost all works use single answer for generation; few exceptions (Jiang et al., 2020c)
- How to guide learning with multiple references: largely open research problem

SECTION 10.3: SELECTION OF TUNING STRATEGY

- Wide variety of methods for tuning prompts, LMs, or both
- Still lack systematic understanding of tradeoffs between methods
- Field could benefit from systematic explorations (cf. Peters et al., 2019 for pre-train/fine-tune paradigm)

SECTION 10.4: MULTIPLE PROMPT LEARNING

Prompt Ensembling:
- Space and time complexity increase with more prompts
- How to distill knowledge from different prompts: underexplored
- Schick & Schütze (2020, 2021a,b): Ensemble model annotates large dataset; final model distills
- How to select ensemble-worthy prompts: under-explored
- For text generation: prompt ensemble learning not performed; ensemble in generation itself is complicated
- Potential: neural ensembling methods like Refactor (Liu et al., 2021c)

Prompt Composition and Decomposition:
- How to choose between them: crucial
- Empirically: Token/span prediction tasks (NER) → prompt decomposition; span relation prediction (entity coreference) → prompt composition
- General idea of de-/composing: can be explored in more scenarios

Prompt Augmentation:
- Limited by input length (too many demonstrations infeasible)
- How to select informative demonstrations and order appropriately: interesting challenge (Kumar & Talukdar, 2021)

Prompt Sharing (multi-task/domain/language):
- Key issues: how to design individual prompts for different tasks; how to modulate interactions
- So far this field has not been explored
- Figure 5: Illustrates simple strategy with partially shared templates

SECTION 10.5: SELECTION OF PRE-TRAINED MODELS

- Many pre-trained LMs to choose from (§3)
- How to select LM to better leverage prompt-based learning: interesting and difficult
- Few/no systematic comparisons of benefits of prompt-based learning across different pre-trained LMs

SECTION 10.6: THEORETICAL AND EMPIRICAL ANALYSIS

- Theoretical analysis and guarantees: scarce
- Wei et al. (2021): Soft-prompt tuning relaxes non-degeneracy assumptions needed for downstream recovery (extraction of task-specific information)
- Saunshi et al. (2021): Verified that text classification tasks can be reformulated as sentence completion tasks, making language modeling a meaningful pre-training objective
- Scao & Rush (2021): Empirically show prompting is often worth 100s of data points on average across classification tasks

SECTION 10.7: TRANSFERABILITY OF PROMPTS

- Understanding extent to which prompts are model-specific and improving transferability: important
- Perez et al. (2021):
  - Tuned few-shot learning (larger validation set): Prompts generalize well across similar-sized models
  - True few-shot learning (only a few samples): Prompts do NOT generalize effectively
  - Transferability is poor when model sizes differ significantly in both scenarios

SECTION 10.8: COMBINATION OF DIFFERENT PARADIGMS

- Much prompt-based success built on pre-trained models from pre-train/fine-tune paradigm (e.g., BERT)
- Open question: Are pre-training methods effective for fine-tuning also effective for prompting as-is?
- Or should pre-training methods be re-designed entirely for prompting-based learning?
- Important research question; not extensively covered in literature

SECTION 10.9: CALIBRATION OF PROMPTING METHODS

- Calibration: ability of model to make good probabilistic predictions
- Jiang et al. (2020b): Observed BART, T5, GPT-2 probabilities on QA tasks are well calibrated
- Zhao et al. (2021): Three calibration pitfalls with answered prompts:
  1. Majority label bias: Model biased toward majority label in examples
  2. Recency bias: Model biased toward answer from most recent example
  3. Common token bias: Model biased toward common tokens
- Zhao et al. (2021) solution:
  1. Use context-free input (e.g., "Input: N/A. Sentiment:") → get initial distribution P0
  2. Use real input → get distribution P1
  3. Combine P0 and P1 for calibrated distribution
  - Drawbacks: (1) overhead finding proper context-free input; (2) underlying LM still uncalibrated

- Holtzman et al. (2021): Surface form competition problem
  - All surface forms of same object compete for finite probability mass
  - Example: "Whirlpool bath" has low probability because "Bathtub" takes probability mass
  - Solutions: (i) Answer engineering with comprehensive gold answer set via paraphrasing; (ii) Calibrate word probability based on prior likelihood in context

SECTION 11.2: TREND ANALYSIS

Year:
- Prompt-based learning increasingly active with emergence of pre-trained LMs
- Huge surge in 2021, likely due to GPT-3 popularizing prompting in few-shot multi-task setting

Tasks (Figure 6b):
- Most common: Text Classification (TC) and Factual Probing (FP)
- Likely because: template/answer engineering is easier; experiments computationally inexpensive

Prompt vs. Answer Search (Figure 6c):
- Current research mainly focuses on template search rather than answer search
- Reasons:
  1. For generation tasks (summarization, translation): gold references can be used directly as answers; multi-reference learning is non-trivial
  2. For classification: label words are relatively easy to select via domain knowledge

Discrete vs. Continuous Search (Figure 6d):
- Shift over time from discrete to continuous search for prompt engineering
- Reasons:
  1. Discrete search harder to optimize
  2. Soft prompts have greater representation ability

A.2: AUXILIARY OBJECTIVES

- Next Sentence Prediction (NSP) [Devlin et al., 2019]: Binary classification — do two segments appear consecutively or are they random?
- Sentence Order Prediction (SOP) [Lan et al., 2020]: Binary — are two sentences in natural or swapped order?
- Capital Word Prediction (CWP) [Liu et al., 2020b]: Binary — is each word capitalized?
- Sentence Deshuffling (SDS) [Liu et al., 2020b]: Multi-class — reorganize permuted segments
- Sentence Distance Prediction (SDP) [Liu et al., 2020b]: Three-class — adjacent in same doc / not adjacent but same doc / different docs
- Masked Column Prediction (MCP) [Yin et al., 2020]: Given table, recover names and data types of masked columns
- Linguistic-Visual Alignment (LVA) [Lu et al., 2019]: Binary — can text content be aligned to visual content?
- Image Region Prediction (IRP) [Su et al., 2020]: Given image with masked features, predict masked regions
- Replaced Token Detection (RTD) [Xiao et al., 2021]: Binary — was each token replaced by generative sample or not?
- Discourse Relation Prediction (DRP) [Sun et al., 2020]: Predict semantic/rhetorical relation between two sentences
- Translation Language Modeling (TLM) [Lample & Conneau, 2019]: Parallel sentences; randomly mask words in both source and target
- Information Retrieval Relevance (IRR) [Sun et al., 2020]: Predict information retrieval relevance of two sentences
- Token-Passage Prediction (TPP) [Liu et al., 2020b]: Identify keywords of passage appearing in segment
- Universal Knowledge-Text Prediction (UKTP) [Sun et al., 2021]: Incorporate knowledge into one pre-trained LM
- Machine Translation (MT) [Chi et al., 2021a]: Translate source language sentence to target
- Translation Pair Span Corruption (TPSC) [Chi et al., 2021a]: Predict masked spans from a translation pair
- Translation Span Corruption (TSC) [Chi et al., 2021a]: Mask and predict spans in one language only

A.2 ADDITIONAL AUXILIARY OBJECTIVES (continued):

- Multilingual Replaced Token Detection (MRTD) [Chi et al., 2021b]: Distinguish real input tokens from corrupted multilingual sentences via GAN; generator and discriminator shared across languages
- Translation Replaced Token Detection (TRTD) [Chi et al., 2021b]: Distinguish real and masked tokens in translation pair via GAN
- Knowledge Embedding (KE) [Wang et al., 2021]: Encode entities and relations in knowledge graphs as distributed representations
- Image-to-text transfer (ITT) [Wang et al., 2021]: Generate description for input image (similar to image captioning)
- Multimodality-to-text transfer (MTT) [Wang et al., 2021]: Generate target text based on both visual and noised linguistic information
