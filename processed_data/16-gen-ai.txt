PAGE 4 — REINFORCEMENT LEARNING (DEFINITION)
================================================================================
Definition: Reinforcement learning (RL) is a field of machine learning that aims to train an agent to perform optimally within a given environment, with respect to a particular goal.

Key RL components (from diagram):
- AGENT: the entity that takes actions
- ENVIRONMENT: the world the agent operates in
- State (s ∈ S): current situation
- Action (a ∈ A): move taken by agent
- Reward (r): value returned by environment after action
- New State (s' ∈ S): state after action

Diagram 1 description: Agent interacts with Environment. Agent takes action, receives reward r, transitions to new state s'.
Diagram 2 description: Loop — state and reward go into Agent; Agent outputs action; action goes into Environment; Environment feeds back state and reward.

================================================================================
PAGE 5 — ENVIRONMENT
================================================================================
Definition: The environment is the world in which the agent operates. It defines the set of rules that govern the game state update process and reward allocation, given the agent's previous action and current game state.

Example — Chess:
- Environment = rules governing how an action (e.g., pawn move e2e4) affects the next game state (new positions of pieces on board).
- Specifies how to assess if a given position is checkmate.
- Allocates a reward of 1 after the winning move.

================================================================================
PAGE 6 — OTHER ELEMENTS (GLOSSARY)
================================================================================
Agent: The entity that takes actions in the environment.

Game State (also just "state"): The data representing a particular situation the agent may encounter. Example: a particular chessboard configuration with accompanying game information such as which player will make the next move.

Action: A feasible move that an agent can make.

Reward: The value given back to the agent by the environment after an action has been taken. The agent aims to maximize the long-term sum of its rewards. Examples:
  - Chess: checkmate opponent's king = reward of 1; every other move = reward of 0.
  - Space Invaders: rewards constantly awarded throughout the episode (points).

Episode: One run of an agent in the environment. Also called a "rollout."

Timestep: For a discrete event environment, all states, actions, and rewards are often subscripted to show their value at timestep t.

PAGE 7 — HOW THE AGENT LEARNS (RL LOOP)
================================================================================
Question posed: "We know about all the elements. But how does the agent learn?"

Learning loop sequence:
1. Start with a state s₀
2. Decide what action to take
3. Execute the action
4. Get the reward
5. Observe the environment (new state s₁)
6. Return to action selection
7. Repeat...

PAGE 11 — MASTER ARCHITECTURE DIAGRAM
================================================================================
Title: THE MASTER ARCHITECTURE

Caption: Observation becomes Representation (z). Representation and Memory (h) dictate Action. Action and Representation update Memory.

Flow description:
- ENVIRONMENT → sends Observation → VAE (V)
- VAE (V) → encodes observation into latent vector z
- z → feeds into MDN-RNN (M) and CONTROLLER (C)
- MDN-RNN (M) → produces hidden state h
- h → feeds into CONTROLLER (C)
- CONTROLLER (C) → outputs ACTION (red arrow)
- ACTION → feeds back into ENVIRONMENT and MDN-RNN (M)

Component roles:
- VAE (V): perception / vision component — compresses raw observation into z
- MDN-RNN (M): memory / foresight component — maintains temporal context h
- CONTROLLER (C): decision-making component — maps z + h to actions

PAGE 14 — STEP 2: TRAINING THE VAE (COMPRESSION)
================================================================================
Title: Step 2: Training the VAE (Compression)
Mini-map position: VAE (V) [highlighted]

Description: The Variational Autoencoder condenses 12,288 raw pixels into just 32 numbers. This latent vector (z) contains the distilled essence of the scene — the straightness of the road, the sharpness of upcoming bends, and the car's position.

Architecture:
- Input: 64×64×3 image = 12,288 pixels
- Process: 4 convolutional layers (Conv Layer 1 → Conv Layer 2 → Conv Layer 3 → Conv Layer 4)
- Output: 32-Dimensional Latent Vector (z) = "Distilled Essence of Scene"

Diagram description: Funnel-shaped convolutional compression — wide input image narrows through successive conv layers to produce the compact latent vector z. Inset shows Encoder → z (latent) → Decoder structure with x (input), h (latent), x' (reconstruction).

PAGE 17 — HANDLING UNCERTAINTY: THE MIXTURE DENSITY NETWORK (MDN)
================================================================================
Title: Handling Uncertainty: The Mixture Density Network
Mini-map position: MDN-RNN (M) [highlighted]

Description: The future is probabilistic. The MDN doesn't predict one definitive next state; it models multiple potential outcomes. It outputs parameters for five overlapping normal distributions, allowing the agent to anticipate varied outcomes and avoid catastrophic overfitting.

Output parameters:
- logpi (mixing coefficients) — blue
- mu (means) — green
- logvar (log variances) — red
- Reward — yellow/gold

Formula: 5 mixtures × 3 parameters × 32 dimensions + 1 reward = 481 Parameters

Diagram description: A car moving forward with multiple overlapping probability distribution "lobes" (blue and gold/tan) projecting ahead of it — representing the multiple possible future states the MDN models.

Key insight: Instead of predicting one deterministic next state, MDN outputs a probability distribution over possible next states — critical for handling the inherent uncertainty of real-world dynamics.

PAGE 18 — STEP 4: THE CONTROLLER (REFLEXES)
================================================================================
Title: Step 4: The Controller (Reflexes)
Mini-map position: CONTROLLER (C) [highlighted]

Description: The actual decision-making brain is astonishingly simple. It is a densely connected network with zero hidden layers. It maps the current vision (z) and the predictive memory (h) directly to physical actions.

Architecture:
- Inputs: z (length 32) + h (length 256) = 288 Total Inputs
- Network: fully connected, no hidden layers (single-layer linear network)
- Outputs: 3 action values:
  1. Turn (steering)
  2. Accelerate
  3. Brake

Parameter count: 288 inputs × 3 outputs = 864 weights + 3 biases = 867 Total Parameters

Diagram: 288 input nodes (green = z, blue = h) all connect directly to 3 output nodes (red circles: Turn, Accelerate, Brake).

Key insight: Despite the controller being the "decision brain," it is intentionally kept minimal (linear, no hidden layers) because all the complex representation is already encoded in z and h.

PAGE 19 — STEP 5: EVOLUTIONARY TRAINING (CMA-ES)
================================================================================
Title: Step 5: Evolutionary Training (CMA-ES)
Mini-map position: CONTROLLER (C) [highlighted]

Context: This is a reinforcement learning problem — there is no labeled data of what a "good" action is — therefore supervised learning cannot be used.

Algorithm: Covariance Matrix Adaptation Evolution Strategy (CMA-ES)
Goal: Evolve the optimal 867 controller parameters over generations.

CMA-ES Loop (4 stages):
1. Spawn Population → Randomly initialize parameters
2. Evaluate in Environment → Test average reward
3. Breed Best Performers → Combine top 25%
4. Add Randomness & Update → Generate next generation
→ Loop back to Spawn Population

PAGE 20 — COVARIANCE MATRIX ADAPTATION VISUALIZED
================================================================================
Title: Covariance Matrix Adaptation Visualized

Description: The algorithm evaluates a population of agents, identifies the top performers, and mathematically shifts the "search area" toward their parameters. With each generation, the covariance matrix adapts, naturally creating momentum toward the optimal driving strategy without calculating costly gradients.

Visualization — 3 stages shown:
- Generation 1: Large, diffuse, oval-shaped distribution of candidate solutions (black dots) spread across parameter space. Search area (dashed ellipse) is wide and not well-aligned with the optimum.
- Generation 3: Distribution has rotated and elongated, narrowing toward the region of higher performance (brighter/yellow zones on contour map).
- Generation 6: Distribution has collapsed to a tight cluster near the optimal parameter region (center of the bright yellow target zone).

Key insight: CMA-ES doesn't use gradients. It learns the shape (covariance) of good solutions and adapts the search distribution accordingly, generation by generation.

================================================================================
Title: The Synthesis: In-Dream Training

Description: The MDN-RNN has learned the physics of the world so perfectly that the real environment can be replaced entirely. The agent feeds an action to its own memory, which hallucinates the next latent state (z) and reward. The agent learns to drive purely by thinking about driving.

Key change: The CarRacing (OpenAI Gym) Environment is CROSSED OUT — eliminated from the training loop.

Dream Environment components and flow:

- MDN-RNN (M): receives action from Controller → outputs next z → feeds z to VAE Decoder; also maintains recurrent h loop (blue arrows)
- VAE (V) Decoder: receives z from MDN-RNN → decodes to visual representation
- MDN (M): outputs sample (probability distribution)

Key insight: The MDN-RNN acts as a generative world model — it simulates the entire physics of the environment internally, allowing training without ever touching the real environment.

PAGE 24 — CONTROLLING THE DREAM (VOLATILITY & TEMPERATURE)
================================================================================
Title: Controlling the Dream (Volatility & Temperature)

Problem: An overly predictable dream creates an overconfident agent (overfitting to the dream).

Solution: Introduce a temperature parameter (tau) when sampling z, forcing the hallucination to be slightly more chaotic than reality.

Temperature dial: tau = 1.15 (highlighted as optimal)

Results table:

| Temperature tau | Virtual Score  | Actual Score   |
|-----------------|----------------|----------------|
| 0.10            | 2086 ± 140     | 193 ± 58       |
| 0.50            | 2060 ± 277     | 196 ± 50       |
| 1.00            | 1145 ± 690     | 868 ± 511      |
| 1.15 ★          | 918 ± 546      | 1092 ± 556     |
| 1.30            | 732 ± 269      | 753 ± 139      |

Key observations:
- At low tau (0.10, 0.50): Virtual score is high but actual score is very low → severe overfitting to dream; agent fails in reality.
- At tau = 1.15: Virtual score and actual score are closely aligned; actual score = 1092 ± 556, which is a SUPERHUMAN score.
- At tau = 1.30: Both scores drop, suggesting too much chaos degrades learning.

Key insight: tau = 1.15 is the sweet spot — slightly more chaotic than reality — enabling the agent to generalize from the dream back to the real world with superhuman performance.

PAGE 25 — THE WORLD MODELS PARADIGM (SUMMARY)
================================================================================
Title: The World Models Paradigm

Description: By combining generative models with evolutionary strategies, agents are no longer bound by the speed of their environment. They can observe, build an internal universe, and master complex tasks through the sheer speed of thought.

Three pillars:

1. CONDENSE REALITY (green)
   - Component: VAE
   - Function: Extracts the latent z
   - Analogy: Compresses high-dimensional sensory input into compact, meaningful representation

2. SIMULATE THE FUTURE (blue)
   - Component: MDN-RNN
   - Function: Hallucinates probabilities
   - Analogy: Predicts multiple possible future states, enabling planning without real-world interaction

3. EVOLVE THE SOLUTION (red)
   - Component: CMA-ES
   - Function: Optimizes reflexes purely in the mind
   - Analogy: Evolutionary optimization of the controller using only the dream environment — no real environment needed

Overall paradigm: Generative model (VAE + MDN-RNN) + Evolutionary optimizer (CMA-ES) = agent that learns entirely inside its own hallucinated reality, then transfers that learning to the real world.
