PAGE 3 - A Brief Summary of ML Terminologies (1)
--------------------------------------------------------------------------------
Section: A Brief Summary of ML Terminologies (Part 1)

Key concept: Set of objects X that we want to label.
Examples:
  - Set of students → predict whether they will get an A in class
  - Set of X-ray images → predict whether the person has a bone disease
  - Set of images → predict whether the image is a car

Label set Y:
  - 0 or 1 (Boolean – discrete)
  - Set of labels {A, B, …} (categories - discrete)
  - Real number (continuous)

Training data:
  - Notation: S = ((x1,y1), (x2,y2), … , (xn,yn))
  - Formally: a subset of X × Y (Cartesian product of X and Y)
  - Meaning: (xi, yi) means xi has the label yi

PAGE 4 - A Brief Summary of ML Terminologies (2)
--------------------------------------------------------------------------------
Section: A Brief Summary of ML Terminologies (Part 2)

Output of an ML algorithm: prediction rule (model)
  - Function h: X → Y (also called: hypothesis, predictor, classifier)
  - Used to calculate the label y of a piece of data x (i.e., h(x))
  - A(S) denotes the output of algorithm A given training dataset S

Data-generation model (assumptions):
  - Data is generated by some probabilistic distribution D
  - Labels are assigned by a labeling function f
  - IMPORTANT: The algorithm does NOT know D and f

Measuring success:
  - Many algorithms, several training data sets → need to compare
  - Define: generalization error (also called: true error, risk) of h → denoted as L (the loss)
  - Goal: algorithms must minimize L(h)

Empirical Risk Minimization (ERM):
  - True error L(h) cannot be directly established
  - Use training set error LS(h) as an approximation
  - Risk: using ERM can lead to overfitting

Overfitting:
  - Model h performs exceptionally on training set S but poorly on unseen data
  - Solution: identify classes of training sets S so that overfitting does not occur

PAGE 7 - Keras API – Activation Functions
--------------------------------------------------------------------------------
Section: Keras API – Activation Functions

1. ReLU (Rectified Linear Unit) — also variant: LeakyReLU
   Formula: ReLU(x) = x⁺ = max(0, x) = (x + |x|) / 2
   Piecewise: x if x > 0, else 0
   Reference: https://en.wikipedia.org/wiki/Rectified_linear_unit

2. Sigmoid
   Formula: σ(x) = 1 / (1 + e^(−x)) = e^x / (1 + e^x) = 1 − σ(−x)
   Reference: https://en.wikipedia.org/wiki/Sigmoid_function

3. Softmax
   Formula: σ(z)_i = e^(z_i) / Σ(j=1 to K) e^(z_j)
   Use case: multi-class classification (outputs probability distribution over K classes)

Definition: The optimizer is the algorithm that adjusts the weights of neurons to minimize the error (loss function).

PAGE 10 - Convolutional Neural Network (CNN)
--------------------------------------------------------------------------------
Section: Convolutional Neural Network

Purpose: CNNs were introduced specifically to work with image data.

How convolution works:
  - A filter (kernel) slides over a portion of the image
  - Element-wise multiplication followed by summation
  Example:
    3×3 image patch * filter = 2.8 (first example)
    3×3 image patch * filter = -0.1 (second example)

CNN layer structure example (from diagram):
  - Input layer: 1 × 64 × 64 × 1 (batch_size × height × width × channels)
  - 2 filters, each 3×3×1
  - Output: 1 × 64 × 64 × 2 (batch_size × height × width × channels)

Practical reference: See convolutions.ipynb for hands-on demonstration.

PAGE 11 - Stride & Padding
--------------------------------------------------------------------------------
Section: Stride & Padding

Stride:
  - Defines the step size (jump) by which the filter moves across the image
  - Larger stride → smaller output feature map

Padding:
  - Adds border pixels (usually zeros) around the input image
  - Allows the filter to cover edge pixels
  - Helps control the spatial dimensions of the output

Visual: Diagram shows a filter (green grid) sliding over a padded image (blue grid) in 4 sequential steps.

PAGE 13 - Batch Normalization
--------------------------------------------------------------------------------
Section: Batch Normalization

Problem addressed: Covariate shift — the issue of weights becoming too large during training.

Algorithm (Batch Normalizing Transform applied to activation x over a mini-batch):

Input:
  - Values of x over a mini-batch: B = {x_1 ... x_m}
  - Parameters to be learned: γ (Gamma), β (Beta)

Output:
  - {y_i = BN_{γ,β}(x_i)}

Steps:
  1. Calculate mini-batch mean:
     μ_B ← (1/m) Σ(i=1 to m) x_i

  2. Calculate mini-batch variance:
     σ²_B ← (1/m) Σ(i=1 to m) (x_i − μ_B)²

  3. Normalize (force mean=0, variance=1):
     x̂_i ← (x_i − μ_B) / √(σ²_B + ε)

  4. Scale and Shift (learnable parameters γ and β):
     y_i ← γ * x̂_i + β ≡ BN_{γ,β}(x_i)

Learnable parameters:
  - Gamma (γ): scale
  - Beta (β): shift

PAGE 14 - Dropout Layer
--------------------------------------------------------------------------------
Section: Dropout Layer

Purpose: Prevents overfitting by randomly deactivating neurons during training.

Training time behavior:
  - Randomly drops (zeroes out) selected neuron inputs
  - Rate = 0.5 means 50% of neurons are dropped per forward pass

Testing time behavior:
  - Does NOT drop neurons
  - Instead: calculates the moving average of the mean and standard deviation of each channel
  - Stores this value as part of the layer for use at test time

Example (from diagram, rate = 0.5):
  - Neuron with value 0.6 → dropped → output 0
  - Neuron with value 0.8 → kept → output 0.8
  - Neuron with value 0.2 → kept → output 0.2
  - Neuron with value 1.0 → dropped → output 0
