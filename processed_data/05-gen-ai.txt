PAGE 3 - Autoencoder (Architecture)
================================================================================
Heading: Autoencoder

Architecture:
- Learn efficient codings of unlabeled data using two functions:
  1. Encoding function: takes input data and maps it to a code (x → h)
  2. Decoding function: recreates the input from the code (h → x*)

Key element:
- Reconstruction loss function: the difference between the input (x) and the reconstructed output (x*)

Applications:
- Dimensionality reduction
- Removing noise (data denoising)
- Feature extraction
- Robustness (anomaly detection)

Diagram components:
- Input Layer (x)
- Encoder (compresses input)
- Code / Bottleneck Space / Latent Space (h)
- Decoder (reconstructs output)
- Output Layer (x')

Note: The middle compressed representation is called the Bottleneck Space or Latent Space.

PAGE 9 - The Latent Space (Problem with Standard Autoencoders)
================================================================================
Heading: The Latent Space

Problem with standard autoencoders for generation:
- Can pick any point in latent space and get an image
- However, it is difficult to figure out how to choose a random point in the latent space
- Reason: the distribution of the points is not defined

Key issue:
- Areas between data clusters may produce poor or meaningless outputs
- Decoded embeddings do not cover the space uniformly

Observation from diagram:
- Left side: scattered point cloud (unlabeled latent points)
- Right side (color overlay): shows where actual image classes exist
- Many empty/sparse regions in the latent space produce unusable outputs

Conclusion: "It is working but we want more than that!" → motivates Variational Autoencoders (VAE)

PAGE 10 - Variational Autoencoder (VAE)
================================================================================
Heading: Variational Autoencoder

Purpose:
- Learn to compress data into a compact and organized space
- Use that space to generate brand new, similar data

Main components: two neural networks and a latent space

1. Encoder:
   - Takes input and compresses it
   - Maps it to a probabilistic distribution (not a single point, unlike standard autoencoders)

2. Latent Space:
   - The space where the model stores the most important features of the data as mathematical variables
   - Organized and continuous

3. Decoder:
   - Samples a point from the distribution
   - Attempts to reconstruct the original input as accurately as possible

Diagram:
Input → Encoder (green layers) → Latent Space (red/distribution icon) → Decoder (blue layers) → Output (x')

Key distinction from standard autoencoder:
- Standard AE: encoder outputs a single point in latent space
- VAE: encoder outputs a probability distribution over the latent space

================================================================================
PAGE 11 - The Key Difference – Encoder (Normal Distribution)
================================================================================
Heading: The Key Difference – Encoder

Comparison:
- Autoencoder: encodes image to a single point in the latent space
- Variational Autoencoder: encodes image to a multivariate normal distribution over the latent space

Normal Distribution (Gaussian Distribution) N(μ, σ):
- A probability distribution characterized by a distinctive bell curve shape
- Defined by two variables:
  - Mean (μ): center of the distribution
  - Variance (σ²): spread of the distribution
  - Standard deviation (σ): square root of the variance
- Formula: f(x | μ, σ²) = (1 / √(2πσ²)) * e^(−(x−μ)² / 2σ²)

Examples of different normal distributions (from plot):
- μ = 0, σ² = 0.2 (narrow, tall)
- μ = 0, σ² = 1.0 (standard normal)
- μ = 0, σ² = 5.0 (wide, flat)
- μ = −2, σ² = 0.5 (shifted left, moderate width)

VAE encodes each input as a multivariate normal distribution in the latent space (not a single point).

PAGE 12 - The Encoder (VAE Mechanics)
================================================================================
Heading: The Encoder

VAE Encoder outputs:
- z_mean: mean vector for each latent dimension
- z_log_var: log variance vector (logarithm of the variance of each dimension)

Assumption:
- Variational autoencoders assume no correlation between dimensions in the latent space (diagonal covariance matrix)

Sampling – Reparameterization Trick:
- Formula: z = z_mean + z_sigma * epsilon
- Where:
  - z_sigma = exp(z_log_var * 0.5)
  - epsilon ~ N(0, I)  [sampled from standard normal distribution]
- Purpose: allows backpropagation through the sampling operation

Latent Space property – Continuous:
- Decoder ensures that points in the same neighborhood produce similar images when decoded
- Enables smooth interpolation between concepts

Loss Function (VAE):
- Total Loss = Reconstruction Loss + Kullback-Leibler Divergence (KL Divergence)
- KL Divergence: measures how much one probability distribution differs from another
- KL Divergence formula (for VAE with standard normal prior):
  D_KL[N(μ, σ) || N(0, 1)] = −(1/2) * Σ(1 + log(σ²) − μ² − σ²)

         → z_log_var →

PAGE 17 - Latent Space Arithmetic
================================================================================
Heading: Latent Space Arithmetic

Concept: Attributes can be added or subtracted in the latent space

Formula for modifying a latent vector:
  z_new = z + alpha * feature_vector

Where:
  z = z_mean + z_sigma * epsilon
  z_sigma = exp(z_log_var * 0.5)
  epsilon ~ N(0, I)

Feature vector computation:
  feature_vector (e.g., "smiling") = 
    average position of encoded images WITH the attribute "smiling"
    MINUS
    average position of encoded images WITHOUT the attribute "smiling"

Attributes demonstrated:
- Smiling: subtracting removes smile, adding increases smile
- Black hair: controls hair color (lighter ← → darker)
- Eyeglasses: removes/adds eyeglasses
- Young: controls apparent age
- Male: controls gender presentation
- Blond hair: controls hair color (brown ← → blonde)

Visualization (grid):
- Rows: different attributes (Smiling, Black hair, Eyeglasses, Young, Male, Blond hair)
- Columns: alpha values from -4 (Subtracting vector) to +4 (Adding vector)
  - 0 = Original image
  - Negative alpha = subtract the attribute
  - Positive alpha = add the attribute

PAGE 18 - Morphing
================================================================================
Heading: Morphing

Concept: Interpolate between two images in the latent space

Formula:
  z_new = z_A * (1 – alpha) + z_B * alpha

Where:
  - z_A = latent encoding of Image A
  - z_B = latent encoding of Image B
  - alpha ranges from 0 to 1

Behavior:
  - alpha = 0 → z_new = z_A (Image A)
  - alpha = 1 → z_new = z_B (Image B)
  - Intermediate alpha values → smooth blend between Image A and Image B

Visualization (grid):
- Rows: 6 different A→B image pairs (celebrity faces)
- Columns: 11 alpha steps from 0 (Image A) to 1 (Image B)
- Intermediate columns show smooth morphing transition between faces

Key property enabled by VAE: continuous, smooth latent space allows meaningful interpolation between any two images

Summary of Key Concepts:

1. AUTOENCODER: Unsupervised model with encoder + decoder. Learns to compress then reconstruct data. Latent space is unstructured.

2. VARIATIONAL AUTOENCODER (VAE): Extension of autoencoder. Encoder outputs a distribution (mean + log variance) instead of a point. Latent space is continuous and structured.

3. REPARAMETERIZATION TRICK: z = z_mean + z_sigma * epsilon; allows gradient flow through sampling step.

4. KL DIVERGENCE LOSS: Regularizes latent space toward standard normal distribution. Combined with reconstruction loss.

5. LATENT SPACE ARITHMETIC: Attributes encoded as vectors; can be added/subtracted to manipulate generated images.

6. MORPHING: Linear interpolation between two latent vectors produces smooth visual transitions.

7. DATASETS USED: Fashion-MNIST (clothing), CelebA (200k celebrity faces with attribute labels).

8. KERAS LAYERS USED: Input, Flatten, Dense, Convolutional, Convolutional Transpose (stride=2), Dropout, Reshape.

9. TRAINING CONFIG: Loss = binary_crossentropy + KL divergence; Optimizer = Adam.
